# Learning Content Architecture

## SRS (Spaced Repetition System)
- SM-2 알고리즘 기반 간격 반복 스케줄링
- 각 학습 아이템에 대한 개별 난이도(EF), 반복 횟수, 다음 복습일 추적
- 정답/오답에 따른 간격 동적 조절
- 일일 복습 큐 자동 생성

## 콘텐츠 데이터 모델
- **한자**: 한자, 음독, 훈독, 획수, 획순(SVG path), JLPT 레벨, 부수
- **어휘**: 단어, 읽기(후리가나), 뜻(ko/en), 품사, JLPT 레벨, 예문[], 테마 태그[]
- **문법**: 문법 포인트, 접속 규칙, 의미, JLPT 레벨, 예문[], 유사 문법 비교
- **예문**: 일본어 원문, 후리가나, 한국어 번역, 영어 번역, 오디오 URL

## 스키마 설계 원칙
- JLPT 레벨(N5~N1)은 모든 콘텐츠의 기본 분류 축
- 예문은 별도 테이블로 분리하여 재사용 가능하게 설계
- 학습 진행은 사용자별 `user_progress` 테이블에서 SRS 파라미터 관리
- 태그/테마는 JSONB 또는 다대다 관계로 유연하게 확장
- 한자↔어휘, 문법↔예문 등 다대다 관계 설계

## 학습 세션
- 세션 시작 시 복습 큐 + 신규 아이템 믹스 비율 결정
- 세션 중 정답률 기반 난이도 동적 조절
- 세션 종료 시 학습 통계 업데이트 (시간, 정답률, 진행도)
